# 抖店上架助手 — 产品介绍

## 解决的问题

在抖店上架服装商品，每件商品要在平台上填几十个字段，大量重复劳动。这个工具把上架流程变成：**放图片 → 核对表格 → 点确认**，其余自动完成。

## 工作流程

```
放入商品图片 → AI识图提取属性 → 自动生成30字标题 → 写入分区Excel表格
                                                    ↓
                                   人工核对/修改 → 确认后一键自动填表上架
```

## 核心功能

| 模块 | 做什么 | 用到的技术 |
|------|--------|-----------|
| AI 识图 | 从服装图片识别领型、袖长、颜色、风格等属性 | 通义千问 VL（免费，2000次/天） |
| 标题生成 | 按抖音风格自动生成约30字商品标题 | AI 大模型 |
| 分区表格 | 6个类目各自独立的Sheet，只显示相关字段 | Excel（openpyxl） |
| 浏览器自动化 | 打开抖店后台，逐步填写所有字段 | Playwright |

## 支持的类目

连衣裙 / T恤 / 蕾丝衫 / 针织衫 / 休闲裤 / 半身裙 / 外套 / 时尚套装

## 输入表格长什么样

一个 Excel 文件，10 个 Sheet。前两列固定（参考图/款号），后面按类目各有不同：

```
 列A(红)   列B(黄)   列C(蓝)      列D~N(蓝)        列O~R(黄)
 参考图  |  款号   | 标题~30字 | AI识别属性...   | 售价/规格/备注
```

### 图片文件夹结构

```
商品图/
  参考图/      <-- AI识别用，文件名填在表格列A
  主图1x1/     <-- 1:1主图，按款号命名
  主图3x4/     <-- 3:4主图，按款号命名
```

浏览器自动化根据款号自动从对应文件夹取图上传。

### 10 个 Sheet

| Sheet | 类目 | 特有属性列 |
|-------|------|-----------|
| 1 | 通用配置 | 品牌/发货地/运费等，设一次就行 |
| 2 | 连衣裙 | 领型/袖长/裙长/裙型/腰型 |
| 3 | T恤-Polo衫-衬衫 | 领型/袖长/衣长/版型 |
| 4 | 蕾丝衫-雪纺衫 | 领型/袖长/衣长/版型 |
| 5 | 针织衫-毛衣-羊毛衫 | 领型/袖长/衣长/版型 |
| 6 | 背心-吊带 | 领型/衣长/版型（无袖长） |
| 7 | 休闲裤 | 裤长/裤型/腰型/厚薄 |
| 8 | 半身裙 | 裙长/裙型/腰型 |
| 9 | 外套 | 领型/袖长/衣长/厚薄/衣门襟 |
| 10 | 时尚套装 | 上装领型/袖长 + 下装类型/长度 |

### 列的颜色区分

| 颜色 | 含义 | 对应列 |
|------|------|--------|
| 红色列 | 图片文件名 | A列：参考图 |
| 蓝色列 | AI自动填 | C~N：标题、属性 |
| 黄色列 | 人工填写 | B：款号 + 尾部：售价、规格、备注 |
| 绿色列 | 固定配置 | 通用配置Sheet |

## 自动化覆盖范围

| 页面区域 | 填写内容 | 来源 |
|----------|---------|------|
| 上传页 | 5张1:1主图 | 人工裁剪好的图片文件 |
| 上传页 | 商品标题 | AI生成 |
| 基础信息 | 导购短标题(12字) | AI凝练 |
| 基础信息 | 面料材质 | 固定：聚酯纤维85%+其他15% |
| 基础信息 | 品牌 | 固定：无品牌 |
| 基础信息 | 类目特定属性 | AI识图 → 匹配下拉选项 |
| 图文信息 | 主图(1:1)×5、主图(3:4)×5 | 自动填入 |
| 图文信息 | 主图视频 | 一般都有，没有就不传 |
| 图文信息 | 商品详情（尺码表图+主图填入） | 自动填入 |
| 价格库存 | 发货模式、发货时间 | 按配置填入 |
| 价格库存 | SKU创建+规格图 | 按表格数据自动填 |
| 价格库存 | 尺码固定值 | S:80-95 M:95-105 L:105-115 XL:115-125 |
| 价格库存 | 价格、库存 | 按表格数据填入 |
| 价格库存 | 订单库存计数 | 固定：付款减库存 |
| 服务与履约 | 运费模板、售后、商品状态 | 固定值，无需改动 |

## 不自动做的事

- 图片裁剪（人工更可靠）
- 最终点"发布"（人工审核后手动发布，发布前商品状态为"下架"）

## 理想效果

每件商品从上架到发布仅需 **2~3 分钟**，且每一步都有表格记录留底。
